STEP 3: run command: ./restore_data.sh <path/to/backup/backup.dump>

warning: this will remove the db. Proceed with caution
to remove all volumes and have clean state on the db 

1.Stop and Remove All Containers:
Before removing the volumes, stop and remove all Docker containers to ensure that there are no active connections to the volumes.

docker stop $(docker ps -aq)
docker rm $(docker ps -aq)


2. After stopping and removing all containers, you can remove the unwanted volumes using the docker volume rm command.

docker volume rm cdrrmo-ormoc_postgres-data
docker volume rm cdrrmo-ormoc_postgres-test-data


3. Verify Volume Removal:
Confirm that the volumes are no longer present in the volume directory:

sudo ls -l /var/lib/docker/volumes/
